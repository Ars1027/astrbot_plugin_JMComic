## v0.2.0 - ✨ 搜索、热门榜与大文件上传增强 2026/6/30

### 新增功能
- **分类热门榜** — 新增 `/jm热门 [日|周|月] [分类] [页码]` 指令及 `/jm排行` 别名，支持按时间、分类和页码查询
- **搜索及榜单续页** — 新增直接发送 `更多` 或使用 `/jm更多` 继续显示最近一次搜索、热门榜结果
- **搜索标签自动补全** — API 搜索结果缺少标签时，自动异步获取作品详情并补全当前展示结果
- **NapCat HTTP Stream 上传** — 新增面向百兆文件和跨容器部署的分块群文件上传模式

### 改进
- **搜索展示配置** — 搜索展示数量、多结果标签数量和“更多”缓存时间均可在 WebUI 配置
- **标签展示规则** — 多条搜索结果按配置限制标签数量，单条结果完整显示全部标签
- **自动选择发送链路** — `auto` 模式下小文件使用 Base64，大文件超过阈值后自动切换 HTTP Stream
- **Stream 传输可靠性** — 支持 SHA-256 校验、分片重试、长超时及 NapCat 临时文件自动清理
- **文件发送诊断** — 发送失败时明确返回失败阶段、原始错误和 AstrBot 本地保存路径

### 修复
- **API 搜索多结果没有标签** — 修复移动端 API 搜索结果不携带标签时只能显示 `-` 的问题
- **协议端无法读取本地路径** — 修复 AstrBot 与 NapCat 分容器部署时直接发送本地路径导致 `ENOENT` 的问题
- **百兆文件 WebSocket 超时** — 修复大文件整体 Base64 编码并通过 OneBot WebSocket 上传导致 API 超时的问题
- **失败提示缺少保存路径** — 修复文件超限或上传失败时提示中没有完整本地路径的问题

### 配置与兼容性
- **新增查询配置** — 新增搜索分页、标签限制、续页缓存和标签补全相关配置项
- **新增 Stream 配置** — 新增发送模式、NapCat HTTP 地址、Token、超时、分片大小、保留时间和重试次数配置项
- **新增运行依赖** — 新增 `aiohttp` 依赖，用于 NapCat HTTP Stream 请求
- **已有配置升级** — 已安装用户需将 `file_delivery_mode` 设置为 `auto` 或 `napcat_http_stream` 才会启用 Stream
